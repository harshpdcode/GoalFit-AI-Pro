GoalFit AI — Project Documentation (Part 1)

======================================================================

1. PROJECT TITLE

GoalFit AI — An Intelligent Health & Fitness Management System

A web-based application that leverages rule-based AI algorithms to provide personalized fitness goals, diet plans, workout recommendations, BMI tracking, water intake monitoring, and downloadable health reports — tailored to each user's unique health profile.

======================================================================

2. PROBLEM DEFINITION

In the modern era, lifestyle-related health issues such as obesity, malnutrition, and sedentary behaviour are rapidly increasing. Millions of individuals wish to improve their health but face the following core problems:

  - Lack of Personalization: Generic fitness apps offer one-size-fits-all plans that do not account for individual metrics like BMI, age, gender, activity level, and dietary preference.
  - Information Overload: Users are overwhelmed by conflicting fitness advice from multiple sources with no structured, unified system to follow.
  - No Intelligent Goal Prediction: Most platforms do not predict when a user will realistically achieve their target weight based on their current health parameters.
  - Fragmented Tools: Tracking weight, diet, water intake, and workout progress requires multiple separate apps, leading to poor consistency and user drop-off.
  - No Expert-Level Guidance for Non-Experts: People without access to dieticians or personal trainers have no personalized, data-driven guidance.
  - Zero Administrative Oversight: There is no centralized system for monitoring overall user health trends across a platform, which is crucial for health professionals or gym administrators.

4. CURRENT SYSTEM AND ITS DRAWBACKS

4.1 Existing Systems in the Market

+----------------------+-----------------+-------------------------------------------------+
| System               | Type            | Limitation                                      |
+----------------------+-----------------+-------------------------------------------------+
| MyFitnessPal         | Mobile App      | Requires manual food logging, no AI prediction  |
| HealthifyMe          | Mobile App      | Paid premium for AI features, no offline access |
| Fitbit App           | Wearable-linked | Requires expensive hardware                     |
| Generic Gym Software | Desktop         | No personalization, no diet planning            |
| Excel/Google Sheets  | Manual          | Completely manual, no automation                |
+----------------------+-----------------+-------------------------------------------------+

4.2 Drawbacks of the Current System

1. No Automated Goal Prediction: Existing free tools do not calculate how many weeks it will take to achieve a target weight based on BMI category and weekly change rate.

2. No Integrated Diet + Workout + Water System: Users must use 3–4 separate applications to manage nutrition, exercise, and hydration — there is no unified platform.

3. No BMI-Aware Workout Filtering: Most apps suggest the same exercises regardless of whether a user is Obese, Normal, or Underweight. This can lead to injury.

4. Limited Admin Control: Existing systems provide no backend admin panel to manage meals, workouts, users, and feedback from a centralized dashboard.

5. No PDF Report Generation: Users cannot export a professional health report summarizing their BMI history, diet plan, goal prediction, and step recommendations.

6. High Cost for Personalization: Premium AI-powered plans on platforms like HealthifyMe cost ₹800–₹2000/month, making them inaccessible to most users.

7. No Activity Logging: Existing systems do not log every user action (login, register, feedback) for administrative auditing.

8. First-Login Flow Gaps: Most apps do not force health profile completion on first login, leading to inaccurate recommendations.

5. PROPOSED SYSTEM AND ITS FEATURES

5.1 Overview

GoalFit AI is a Python Flask web application with a MySQL relational database that provides a complete, intelligent, and personalized health management ecosystem accessible via any web browser — without requiring hardware devices or premium subscriptions.

5.2 System Architecture

[User Browser]
     ↓
[Flask Web Server — app.py]
     ↓
[Blueprint Modules: auth, health, bmi, prediction, diet,
 workout, progress, water, feedback, report, admin]
     ↓
[MySQL Database — goalfit_ai]
     ↓
[13 Relational Tables with Foreign Key Constraints]

5.3 Key Features

🔐 Authentication & Role Management
  - Secure user registration with Werkzeug password hashing (bcrypt)
  - Role-based access control:  vs  roles
  - First-login flow: forces health profile completion before dashboard access
  - Session management with Flask sessions
  - Activity logging for every login, logout, and registration event

📋 Health Profile Module
  - Collects: Age, Gender, Height, Weight, Target Weight, Activity Level, Goal Type, Diet Preference
  - Supports both INSERT (new user) and UPDATE (returning user) operations
  - Automatically triggers BMI calculation and AI prediction on save

📊 BMI Calculation Engine
  - Formula: BMI = weight(kg) / (height(m))²- Categories: Underweight (<18.5), Normal (18.5–24.9), Overweight (25–29.9), Obese (≥30)
  - Stores timestamped records for historical tracking

🎯 AI Goal Prediction Engine
  - Uses BMI category + goal type to determine a realistic weekly weight change rate:
  - Weight Loss (Obese): 1.0 kg/week
  - Weight Loss (Overweight): 0.7 kg/week
  - Weight Loss (Normal): 0.4 kg/week
  - Weight Gain (Underweight): 0.6 kg/week
  - Weight Gain (Others): 0.3 kg/week
  - Calculates _weeks = weight_diff / weekly_rate- Computes _completion_date from today
  - Auto-generates daily step target and calorie burn goal based on weight difference

🍽️ Personalized Diet Plan
  - Meals automatically filtered by diet_preference (Vegetarian / Non-Vegetarian / Vegan) and goal_type (Weight Loss / Weight Gain)
  - Organized by meal time: Breakfast, Lunch, Dinner, Snacks
  - Displays: Meal Name, Calories, Proteins, Carbs, Fats, Food Image
  - 40+ pre-loaded meals covering all diet-goal combinations

💪 Smart Workout Plan
  - Exercises filtered by BMI category to set appropriate difficulty:
  - Obese/Overweight → Beginner + Intermediate
  - Normal → Intermediate only
  - Underweight → Intermediate + Advanced
  - Grouped by muscle group: Chest, Legs, Abs, Cardio
  - Each exercise includes: Name, Calories Burned, Difficulty, Image, YouTube Tutorial Link
  - 20 pre-loaded exercises across all muscle groups

📈 Progress Tracker
  - Users log daily weight entries
  - Auto-recalculates BMI, goal prediction, and step recommendations on each log
  - Line chart visualization of weight trend over time
  - BMI trend chart with historical data

💧 Water Intake Tracker
  - Daily water glass counter (default goal: 8 glasses)
  - Add/Remove glass via AJAX (no page reload)
  - Streak counter for consecutive days meeting water goal
  - 7-day historical view
  - Custom daily goal setting

📄 PDF Report Generator
  - One-click downloadable report using ReportLab
  - Report includes: Health Profile, BMI History, Goal Prediction, Step Recommendations, Weight Progress Log, Full Diet Plan
  - Professional branded design with GoalFit AI header

💬 Feedback System
  - Users submit feedback with subject and message
  - View history of past submissions with status (unread/read)
  - Admin can reply and mark status

🛠️ Admin Panel (Complete CMS)
  - Dashboard: Total users, new users this week, average BMI, total meals & exercises, unread feedback count, goal distribution charts, activity logs
  - User Management: View all users as cards, search by name/email, click to view full user detail
  - User Detail View: Full profile, BMI history, progress logs, predictions, activity logs
  - Meal CRUD: Add, Edit, Delete diet meals with all nutritional fields
  - Workout CRUD: Add, Edit, Delete exercises with image/video links
  - Activity Logs: Full audit trail of all user actions (last 100)
  - Feedback Inbox: Filter by status, reply to user feedback
  - Analytics API: JSON endpoint for BMI distribution charts

9. FUNCTIONAL AND NON-FUNCTIONAL REQUIREMENTS

9.1 Functional Requirements

Functional requirements define what the system must do.

FR-01: User Registration
  - The system shall allow new users to register with Name, Email, and Password.
  - The system shall hash the password before storing it.
  - The system shall prevent duplicate email registration.
  - The system shall log the registration event in _logs.

FR-02: User Login / Logout
  - The system shall authenticate users using email and hashed password comparison.
  - The system shall establish a session upon successful login containing _id, _name, , and .
  - The system shall redirect admin users to the admin dashboard.
  - The system shall redirect first-time users to the health profile form.
  - The system shall clear all session data on logout.

FR-03: Health Profile Management
  - The system shall allow users to fill in age, gender, height, weight, target weight, activity level, goal type, and diet preference.
  - The system shall store the profile in _health.
  - The system shall auto-trigger BMI calculation and goal prediction on profile save.

FR-04: BMI Calculation
  - The system shall calculate BMI using  / (height_m)².
  - The system shall categorize BMI as Underweight / Normal / Overweight / Obese.
  - The system shall store each calculation with a timestamp in _records.

FR-05: AI Goal Prediction
  - The system shall calculate estimated weeks to reach target weight.
  - The system shall compute a goal completion date.
  - The system shall generate daily step and calorie burn targets.
  - The system shall store predictions in _predictions and _recommendations.

FR-06: Diet Plan
  - The system shall retrieve meals matching the user's _preference and _type.
  - The system shall display meals grouped by time (Breakfast, Lunch, Dinner, Snacks).
  - The system shall show nutritional values (calories, protein, carbs, fats) for each meal.

FR-07: Workout Plan
  - The system shall retrieve exercises filtered by difficulty based on the user's BMI category.
  - The system shall group exercises by muscle group.
  - The system shall display exercise images and YouTube tutorial links.

FR-08: Progress Tracking
  - The system shall allow users to log daily weight.
  - The system shall automatically recalculate BMI, goal prediction, and step targets on each weight log.
  - The system shall display weight and BMI trend charts.

FR-09: Water Tracking
  - The system shall log daily water intake in glasses.
  - The system shall support add/remove glass via AJAX without page reload.
  - The system shall calculate and display a hydration streak.
  - The system shall display a 7-day history.

FR-10: PDF Report Generation
  - The system shall generate a downloadable PDF report.
  - The report shall include health profile, BMI history, goal prediction, step recommendations, weight log, and diet plan.

FR-11: Feedback System
  - The system shall allow users to submit feedback with subject and message.
  - The system shall display users' own feedback history with status.
  - The admin shall be able to reply and update feedback status.

FR-12: Admin Panel
  - The system shall restrict admin routes using a decorator that verifies  == 'admin' in session.
  - The admin shall be able to view, add, edit, and delete meals and exercises.
  - The admin shall be able to view all users, search by name/email, view detailed user profiles, and delete users.
  - The admin shall be able to view all activity logs and feedback with filters.
  - The admin dashboard shall display aggregate statistics: total users, average BMI, goal distribution, diet distribution, and user growth.

9.2 Non-Functional Requirements

Non-functional requirements define how the system must perform.

+--------+---------------------+------------------------------------------------------------------------------------------------------------------------------------+
| ID     | Category            | Requirement                                                                                                                        |
+--------+---------------------+------------------------------------------------------------------------------------------------------------------------------------+
| NFR-01 | Security        | All passwords must be stored as PBKDF2-SHA256 hashes using Werkzeug. Plain-text passwords must never be stored.                    |
| NFR-02 | Security        | All protected routes must validate session before processing. Unauthorized access redirects to login.                              |
| NFR-03 | Security        | Admin-only routes are protected by an @admin_required decorator. Regular users cannot access admin URLs.                         |
| NFR-04 | Performance     | Dashboard page must load within 2 seconds on a local machine with a standard MySQL query set.                                      |
| NFR-05 | Performance     | PDF report generation must complete within 5 seconds for a standard user data set.                                                 |
| NFR-06 | Reliability     | Database connection failures must be gracefully handled with user-facing error flash messages, not server crashes.                 |
| NFR-07 | Usability       | The UI must be responsive and usable on screens from 768px (tablet) to 1920px (desktop).                                           |
| NFR-08 | Usability       | Flash messages must be displayed for all key user actions (login failure, registration success, form save, etc.).                  |
| NFR-09 | Maintainability | Application must use Flask Blueprints to separate concerns — each module is independently maintainable.                            |
| NFR-10 | Maintainability | Database credentials must be stored in a .env file and never hardcoded in application logic.                                     |
| NFR-11 | Scalability     | The Blueprint-based modular architecture allows new features to be added as independent modules without modifying existing code.   |
| NFR-12 | Data Integrity  | All foreign key relationships must use ON DELETE CASCADE or ON DELETE SET NULL to maintain referential integrity.              |
| NFR-13 | Auditability    | Every significant user action (login, logout, register, admin edits) must be logged in _logs with timestamp and details. |
| NFR-14 | Portability     | The application must run on Windows, Linux, and macOS with no OS-specific dependencies.                                            |
| NFR-15 | Availability    | The system must support concurrent multi-user access without data corruption (handled by MySQL transaction isolation).             |
+--------+---------------------+------------------------------------------------------------------------------------------------------------------------------------+
